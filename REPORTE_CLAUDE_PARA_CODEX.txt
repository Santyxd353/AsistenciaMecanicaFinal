3) BLOQUE B - WEB: BUGS OFFLINE + COTIZACIONES + MOJIBAKE

3.1 Bug 1 - cola offline no sincronizaba al abrir app online
-----------------------------------------------------------
Sintoma: si el usuario creo emergencia offline, cerro la app, y la volvio
a abrir ya con internet, el evento `online` no se disparaba y las
emergencias pendientes quedaban sin sincronizar para siempre.

Fix (offline-queue.service.ts, constructor):

    if (navigator.onLine) {
      this.flush().subscribe();
    }

3.2 Bug 2 - UI cliente quedaba stale tras sincronizar
-----------------------------------------------------
Sintoma: tras reconectar y flush, los reports con estado `pendiente_sync`
y id falso seguian visibles; realtime no se enganchaba al id real.

Fix en 2 partes:
- offline-queue.service.ts: agregado `flushed$: Subject` que emite el
  resultado { synced, pending, failed } al terminar cada flush.
- client-portal.component.ts: se suscribe a `flushed$` en ngOnInit; si
  synced > 0 llama `loadReports()` -> reemplaza los falsos por los reales
  y reengancha realtime al id correcto.

4) BLOQUE C - IA RECONOCIMIENTO DE VEHICULO + IMAGEN BACKEND SLIM
============================================================

4.1 Problema
------------
Codex habia metido en el backend `torch`, `transformers`, `faster-whisper`,
+ system deps (ffmpeg, libgl1, libglib2.0-0, tesseract-ocr, tesseract-spa)
para hacer analisis de vehiculo / damage / audio localmente. La imagen
docker pasaba los 10 GB. El user quiere desplegar en cloud -> esto es
insostenible. Ademas la Gemini free tier dura poco.

Decision (consensuada con el user): TODO IA via HTTP a Groq + Gemini
fallback. Sin ML local. Solo `requests` agregado al backend.

4.2 Nuevo servicio: backend/app/services/groq_client.py
-------------------------------------------------------
Expone:

  - groq_vision_json(image_paths, prompt, max_tokens) -> dict | None
      Chat completions con Llama 4 Scout multimodal. Recibe imagenes,
      devuelve JSON parseado (response_format = json_object).

  - groq_audio_transcribe(audio_path, language) -> dict | None
      audio/transcriptions con whisper-large-v3.

Detalles importantes (DON'T LOSE):

  a) IMAGENES SE CONVIERTEN A JPEG VIA PIL ANTES DE MANDAR.
     Groq vision rechaza algunos formatos (webp dio 403 directo).
     Se redimensiona max 1280px lado mayor (payload chico) y se
     guarda a JPEG quality 85.

  b) USER-AGENT CUSTOM REQUERIDO.
     Cloudflare WAF de Groq devuelve HTTP 403 (error code 1010)
     cuando el User-Agent es `Python-urllib/...` (que es el default
     de urllib). Se manda:
         "User-Agent": "asistencia-mecanica-backend/1.0 (+https://groq.com)"
     Si quitan esto, vuelve a 403. Comprobado.

  c) Logging mejorado: en HTTPError se loguea el body de respuesta
     (primeros 500 chars) -> facilita diagnostico futuro.

  d) Sin GROQ_API_KEY -> devuelve None y deja al caller hacer
     fallback / mostrar mensaje.

4.3 backend/app/services/ai.py - analyze_vehicle_photos reescrito
-----------------------------------------------------------------
Antes: Gemini -> HF model (transformers) -> tesseract local (heuristica).
Ahora:
  1) Intenta Groq vision via _analyze_vehicle_with_groq().
  2) Si Groq no devuelve campos utiles y hay GEMINI_API_KEY, intenta
     Gemini (_analyze_vehicle_with_gemini existente).
  3) Si todo falla -> VehiclePhotoAnalysis con strings vacios y resumen
     "safe-fallback".

Prompt al LLM exige JSON con placa/marca/modelo/anio/color/resumen. Si
no detecta algo con confianza devuelve "" / 0 -> el frontend muestra
"X no identificado" sin bloquear.

Helpers nuevos:
  - _has_useful_fields(result): cualquier campo no vacio.
  - _coerce_anio(value): valida rango 1950-2100.

Stubs (para no romper si algo lo llama):
  - _extract_plate_candidate_text(image) -> []   (tesseract retirado)
  - _extract_candidate_text(image) -> []
  - warm_vehicle_ai() -> None

Top-level `import pytesseract` REMOVIDO (rompia el arranque si se quitaba
la dep). El resto del codigo local sigue ahi como dead code pero no se
llama; si se borra requirements.txt -> el codigo no importa nada al
arrancar.

5.8 Bug pendiente: 401 en /preview-from-photo
---------------------------------------------
En la sesion al final, el user reporto que IA "se queda Analizando...".
Logs backend mostraron:
    "POST /api/v1/vehiculos/preview-from-photo HTTP/1.1" 401 Unauthorized
Diagnostico: el token del cliente expiro / no estaba. Auth interceptor
lee `localStorage.getItem('token')` y agrega Bearer; si no hay token
no agrega header -> backend rechaza.
Solucion temporal: el user debe cerrar sesion y volver a entrar.
Solucion permanente recomendada: implementar refresh_token automatico
en el interceptor cuando llega 401 (el backend ya devuelve refresh_token).

7) COMO LEVANTAR EL STACK EN LOCAL
============================================================
Requisitos: Docker Desktop corriendo, Node instalado, archivo .env en la
raiz (con POSTGRES_*, SECRET_KEY, GROQ_API_KEY).

  # Backend + DB (Docker):

  # Frontend dev (hot reload):
  cd frontend
  npm start            # -> http://localhost:4200

  # Importante: si el contenedor docker `frontend` esta corriendo
  # (restart: always en compose), pelea por el puerto 4200 con ng serve.
  # Se debe detener:
  docker compose stop frontend
