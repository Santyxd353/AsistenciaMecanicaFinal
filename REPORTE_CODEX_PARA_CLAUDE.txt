OBJETIVO ORIGINAL DEL USUARIO
-----------------------------
El sistema debe ser una plataforma tipo Uber/Yango para asistencia vehicular:

- Web Angular para superadmin, talleres, clientes y operadores.
- Backend FastAPI + PostgreSQL.
- App mobile Flutter para clientes, administradores de taller, mecanicos y
  trabajadores.
- SaaS multi-tenant.
- Planes por taller.
- Login separado por tipo de usuario.
- Tracking realtime del mecanico.
- Registro de vehiculos.
- Reporte de emergencias.
- IA para diagnostico: audio, imagen, clasificacion, resumen.
- UX minimalista cafe, basada en pantallas de Stitch.

ESTADO GENERAL
--------------
El proyecto tiene muchos cambios mezclados entre Claude y Codex.
Hay funcionalidades parciales, pantallas redisenadas, servicios nuevos y varios
bugs abiertos.

IMPORTANTE:
Codex NO completo correctamente la IA ni el flujo mobile completo.
El usuario esta molesto con razon porque se mezclaron pantallas tecnicas como
"KPIs" en la app de usuario comun, y algunas pantallas no respetan del todo
